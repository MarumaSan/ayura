# Ayura: Business Feature Summary (For Business Model Canvas)

เอกสารสรุปภาพรวมฟีเจอร์หลัก อัลกอริทึม และจุดขายของระบบ Ayura เพื่อนำไปใช้งานในการทำ Business Model Canvas (BMC) หรือ Pitch Deck

---

## 1. Core Concept (Concept หลักของธุรกิจ)
**Ayura** คือระบบ Ecosystem สำหรับสุขภาพแบบครบวงจร ที่ผสานระหว่าง **"การส่งมอบกล่องวัตถุดิบ (Meal Set) แบบ Subscription"** เข้ากับ **"แอปพลิเคชันจัดตารางอาหารอัจฉริยะ"** ผู้ใช้จะได้รับวัตถุดิบสดใหม่ และแอปจะคอยบอกว่าในแต่ละวันควรนำวัตถุดิบในกล่องมาทำเมนูอะไร เพื่อให้ตรงตามเป้าหมายสุขภาพของตนเอง (โภชนาการ, แคลอรี) โดยที่ของในตู้เย็นไม่เหลือทิ้ง

---

## 2. Main Hero Features (ฟีเจอร์ชูโรง / จุดขายหลัก)

*   **Dynamic "Available-Ingredient" Menu (เมนูแนะนำอิงจากของในกล่อง):** 
    นี่คือ Killer Feature ของเรา ระบบจะไม่แนะนำเมนูมั่วๆ แต่จะแนะนำ **เฉพาะเมนูที่สามารถทำได้จริงจากวัตถุดิบที่มีอยู่ในกล่อง Meal Set ที่ผู้ใช้ซื้อไปเท่านั้น** (Strict Ingredient Matching)
*   **Anti-Fatigue Daily Rotation (ระบบจัดการความจำเจ):** 
    ระบบจะทำการสับเปลี่ยนเมนู (เช้า-กลางวัน-เย็น) ในแต่ละวันแบบอัตโนมัติ โดยมีเงื่อนไขว่า **"เมนูวันนี้ ต้องไม่ซ้ำกับเมื่อวาน"** เพื่อให้ผู้ใช้ไม่เบื่อ แม้จะซื้อวัตถุดิบเซ็ตเดิม
*   **Precision Health Profiling (การประเมินร่างกายแม่นยำสูง):** 
    คำนวณ BMI, BMR, TDEE และเป้าหมายสารอาหาร (Macro: โปรตีน, คาร์บ, ไขมัน) แบบอัตโนมัติจากข้อมูลเชิงลึก (อายุ, ส่วนสูง, น้ำหนัก, เพศ, เป้าหมายสุขภาพ)
*   **Producer Community Tracking (ระบบติดตามแหล่งที่มาฟาร์มชุมชน):** 
    วัตถุดิบทุกชิ้นสามารถระบุกลับไปยัง "ชุมชนแหล่งผลิต" ได้ สร้าง Storytelling แบบ Farm-to-Table สนับสนุนชุมชน และเพิ่มความมั่นใจเรื่องความปลอดภัย
*   **Macro & Calorie Transparency (ความโปร่งใสทางโภชนาการแบบ 100g):** 
    ระบบฐานข้อมูลวัตถุดิบถูกออกแบบให้คำนวณแคลอรีและสารอาหารต่อ 100 กรัมเป็นมาตรฐาน ทำให้การคำนวณโภชนาการรวมของแต่ละสูตรอาหารมีความแม่นยำสูงมาก

## 3. Core Algorithms (อัลกอริทึมที่ขับเคลื่อนระบบ)

1.  **Strict Ingredient Matching Engine:**
    *   อัลกอริทึมที่ทำการ Cross-reference ระหว่าง `Ingredient ID` ทั้งหมดในกล่องของผู้ใช้ กับ `Recipe Ingredients` ในฐานข้อมูล
    *   **Logic:** ถ้าสูตรอาหารไหน มีส่วนผสมแม้แต่ 1 อย่างที่**ไม่มี**ในกล่องของผู้ใช้ สูตรนั้นจะถูกปัดตบทิ้งทันที ทำให้ผู้ใช้มั่นใจว่าเปิดแอปมาแล้ว "ทำตามได้ 100%"
2.  **Deterministic Daily Randomizer (Time-seeded Selection):**
    *   อัลกอริทึมที่ใช้สร้างเมนูประจำวัน ออกแบบมาเพื่อลดภาระเซิร์ฟเวอร์และทำให้ UX คงที่
    *   **Logic:** ใช้ "วันที่ปัจจุบัน (Thai Time)", "Meal Set ID", และ "ประเภทมื้อ (เช้า/กลางวัน/เย็น)" มาสร้างเป็น Hash Seed เพื่อสุ่มเมนู
    *   **ผลลัพธ์:** ผู้ใช้เปิดแอปเช้า สาย บ่าย เย็น เมนูก็ยังคงเป็นเมนูเดิมของวันนั้นๆ แต่จะเปลี่ยนไปโดยอัตโนมัติในเที่ยงคืนถัดไป พร้อมระบบตัดเมนูที่ซ้ำกับเมื่อวานออก
3.  **Automated Subscription Lifecycle & Order States:**
    *   ระบบติดตามสถานะกล่องอาหาร (Pending -> Delivering -> Delivered) 
    *   **Logic:** นำร่องจากการกด "จัดส่งสำเร็จ" ของ Admin ระบบจะนับถอยหลังอายุของกล่อง (เช่น 7 วัน หรือ 30 วัน) และเริ่มรันระบบแสดงเมนูอาหารให้ผู้ใช้ดูทันทีที่กล่องถึงบ้าน

## 4. Selling Points for BMC (จุดขายที่ใช้สื่อสารกับลูกค้า)

*   **Zero Brainpower Required (ไม่ต้องคิดเมนูเองอีกต่อไป):** แก้ปัญหาโลกแตก "วันนี้กินอะไรดี?" แถมมั่นใจได้ว่าสิ่งที่ทำกินนั้นตรงกับเป้าหมาย Diet ของตัวเอง
*   **Zero Food Waste (บอกลาตู้เย็นเน่า):** เพราะสูตรอาหารถูกออกแบบมาให้ Match กับปริมาณของในกล่อง วัตถุดิบจะถูกใช้หมดพอดี ไม่มีผักชีเหลือทิ้ง
*   **Cooking as Self-Care (การทำอาหารคือการเยียวยา):** เราตัดส่วนที่น่าเบื่อ (คิดเมนู, ไปตลาด, คำนวณแคล) ออกไป เหลือไว้แต่ความสนุกในการเข้าครัวปรุงอาหารสดใหม่
*   **High Customization (ยืดหยุ่นสูง):** รองรับสาย Keto, สาย สร้างกล้ามเนื้อ (Muscle Building), หรือสาย Balance โดยไม่ต้องเปลี่ยนแอป แค่เปลี่ยนเซ็ตวัตถุดิบ

---

## 5. System Architecture / Resources (ทรัพยากรหลักของธุรกิจ)
*   **Frontend (User App):** Dashboard สำหรับดูเมนูรายวัน, หน้า Profile สำหรับ Track ร่างกาย, ระบบ Checkout/Subscription, ระบบจัดการ Profile 
*   **Backend (Admin App):** ระบบจัดการวัตถุดิบ/เซ็ตอาหาร/สูตรอาหาร, ระบบจัดการ Orders และอัปเดตสถานะการจัดส่ง, ระบบจัดการ Producer Community
*   **Database (Supabase):** โครงสร้าง Relational Database ที่ผูกข้อมูลวัตถุดิบเข้ากับสูตรอาหารและกล่องแบบ Real-time
